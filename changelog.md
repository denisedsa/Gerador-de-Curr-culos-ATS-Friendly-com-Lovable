# Changelog

Todas as mudanças relevantes deste projeto serão documentadas neste arquivo.

O objetivo é registrar a evolução do **Gerador de Currículos ATS Friendly com Lovable**, destacando novas funcionalidades, melhorias e correções realizadas durante o desenvolvimento.

---

# [1.0.0] - Concepção do Projeto

## Adicionado

* Definição do objetivo da aplicação.
* Levantamento dos requisitos funcionais.
* Estrutura inicial do projeto.
* Planejamento da experiência do usuário (UX).

---

# [1.1.0] - Cadastro Manual

## Adicionado

* Formulário para criação manual de currículos.
* Campos para dados pessoais.
* Campos para resumo profissional.
* Campos para experiências profissionais.
* Campos para formação acadêmica.
* Campos para habilidades.
* Campos para certificações.
* Atualização automática do preview do currículo.

---

# [1.2.0] - Importação de Currículo

## Adicionado

* Importação de currículo existente.
* Extração automática das informações.
* Preenchimento automático do formulário.
* Geração do preview após a importação.

---

# [1.3.0] - Histórico de Currículos

## Adicionado

* Histórico de currículos criados.
* Registro da data de criação.
* Registro da última atualização.
* Edição de currículos salvos.
* Exclusão de currículos.

## Corrigido

* Exibição correta do nome do candidato no histórico.
* Atualização automática da lista após salvar alterações.

---

# [1.4.0] - Exportação ATS Friendly

## Adicionado

* Exportação em PDF.
* Layout otimizado para sistemas ATS.
* Organização das seções para leitura automatizada.

## Corrigido

* Exportação considerando apenas o currículo.
* Ajustes no layout para impressão.

---

# [1.5.0] - Melhorias de Interface

## Melhorado

* Navegação entre as telas.
* Organização visual do formulário.
* Atualização do preview em tempo real.
* Melhorias na experiência do usuário.

## Corrigido

* Ajustes na visualização do histórico.
* Correções de layout.
* Pequenas melhorias de usabilidade.

---

# [1.6.0] - Documentação

## Adicionado

* README do projeto.
* Roadmap.
* Changelog.
* Documentação das regras de negócio.
* Fluxograma do sistema.
* Documentação dos prompts utilizados.

---

## Convenções

Este projeto utiliza a seguinte classificação para registrar alterações:

* **Adicionado** → Novas funcionalidades.
* **Melhorado** → Aprimoramentos em funcionalidades existentes.
* **Corrigido** → Correções de bugs e comportamentos inesperados.
* **Removido** → Funcionalidades descontinuadas.
